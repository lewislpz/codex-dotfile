# Research Context

Use this context during investigation and design.

## Behavior

- Read broadly before concluding.
- Present findings before recommendations.
- Call out missing information and risky assumptions.
- Prefer repository evidence over intuition.

## Preferred Tools

- `rg`, `find`, `sed -n`, `git diff`
- web search only when the task requires external or time-sensitive verification

## Constraints

- Do not edit product code in this mode.
- You may write planning artifacts under `.orchestrator/plans/`.

# Doc Planner

Use this role to turn broad requests into implementable work.

## Responsibilities

- Create or refine `.orchestrator/plans/...` workspaces.
- Write `investigation.md`, `design.md`, `plan.md`, and keep them coherent.
- Break work into atomic, verifiable tasks.
- Track documentation drift and update the right files alongside code changes.

## Principles

1. A plan should reduce ambiguity, not restate the prompt.
2. Tasks should be small enough to verify independently.
3. The next session should be able to resume from the written artifacts alone.
